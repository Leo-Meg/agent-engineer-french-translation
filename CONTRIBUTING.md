# Contribuer à la formation Agent Engineer

Merci de votre intérêt pour l'amélioration de cette formation. Voici comment vous pouvez aider.

## Comment contribuer

Si vous comptez utiliser l'IA pour contribuer, merci de suivre les recommandations ci-dessus et de le préciser clairement dans la description de votre PR (pull request). Voici quelques façons concrètes de contribuer :

- **Corriger les fautes de frappe et les erreurs** - Vous avez repéré une erreur ? Ouvrez une PR avec le correctif.
- **Améliorer les explications** - Si quelque chose prête à confusion, proposez une manière plus claire de l'expliquer.
- **Ajouter des liens** - Vous connaissez un bon codelab, un tutoriel ou une page de documentation pertinente ? Ajoutez-la.
- **Signaler des problèmes** - Quelque chose est obsolète ou cassé ? Ouvrez une issue (un ticket de suivi).

## Recommandations

- Gardez un ton conversationnel et accessible. Cette formation s'adresse à des débutants.
- Utilisez des analogies lorsqu'elles aident à comprendre. Évitez le jargon sans l'expliquer.
- Utilisez « - » plutôt que des tirets cadratins.
- N'utilisez que des caractères simples - pas d'Unicode spécial ni d'emoji.
- Renvoyez vers la documentation officielle plutôt que de dupliquer des références d'API ou des exemples de code.
- Gardez des PR ciblées. Un seul changement par PR est plus simple à relire.

## Ce que nous ne recherchons pas

- Des exemples de code complets ou des tutoriels. Nous renvoyons vers des ressources officielles pour cela.
- Du contenu qui duplique la documentation Google Cloud existante.
- Un langage marketing ou du contenu promotionnel.

## Processus

1. Créez un fork du dépôt
2. Créez une branche pour votre modification
3. Effectuez vos modifications
4. Ouvrez une PR avec une description claire de ce que vous avez changé et pourquoi

## Code de conduite

Soyez bienveillant, respectueux et serviable. Nous sommes tous ici pour apprendre.
